# Korean PDF Writer

![Korean PDF Writer 미리보기](assets/hero.png)

문서 목적과 독자에 맞춰 어려운 한국어 기술 문서를 쉽게 재구성하고, 내용 기반 SVG 다이어그램과 함께 단정한 PDF로 출력하는 오픈소스 웹 도구입니다.

## 특징

- 문서 유형, 독자, 톤을 먼저 묻는 작성 흐름
- 변환 방식 선택
  - 쉽게 재구성: 원문을 요약, 핵심 개념, 쉬운 비유, 이해 순서, 체크리스트로 재편집
  - 원문 유지: 입력한 마크다운 구조를 그대로 인쇄 미리보기로 렌더링
- 제목, 문단, 숫자 목록, 불릿 목록을 단순 마크다운 문법으로 변환
- 내용 키워드에 맞춘 SVG 다이어그램 자동 삽입
  - 프로세스
  - 타임라인
  - 비교
  - 구조 개요
- 한국어 장문에 맞춘 세리프 기반 인쇄 미리보기
- 브라우저 인쇄 기능을 통한 PDF 저장
- 별도 서버 저장 없이 정적 HTML, CSS, JavaScript로 동작

## 사용 방법

1. 문서 유형을 선택합니다.
2. 읽는 사람과 문서 톤을 입력합니다.
3. 변환 방식을 선택합니다.
4. SVG 방식을 자동 또는 특정 유형으로 선택합니다.
5. 원문이나 마크다운 초안을 붙여 넣습니다.
6. 미리보기를 확인한 뒤 **PDF로 저장**을 누릅니다.

## 로컬 재작성 엔진

기본 버전은 API 키 없이 브라우저 안에서 동작합니다. 원문을 서버로 보내지 않고 다음 규칙으로 문서를 재구성합니다.

- 반복되는 용어를 핵심 개념 후보로 추출
- 긴 문장을 중요한 문장 후보로 선택
- 섹션 제목을 이해 순서로 변환
- 독자와 톤 선택값을 문서 상단에 반영
- 본문 키워드에 따라 SVG 유형을 고름

이 방식은 완전한 LLM 요약이 아니라 공개 오픈소스 기본값입니다. 더 자연스러운 설명이 필요하면 아래 AI 확장 프롬프트를 사용자가 직접 연결할 수 있습니다.

## 선택형 AI 확장 프롬프트

사용자가 자기 API 키와 백엔드를 연결할 때 사용할 수 있는 프롬프트 예시입니다. 이 레포는 기본적으로 API 키를 포함하지 않습니다.

```text
너는 한국어 전문 문서 편집자다. 아래 원문을 독자와 목적에 맞춰 쉽게 재구성하라.

문서 유형: {{document_type}}
읽는 사람: {{audience}}
문서 톤: {{tone}}
SVG 방식: {{diagram_mode}}

출력은 다음 마크다운 구조를 따른다.
# {{title}} 쉽게 이해하기
## 이 문서는 누구를 위한 것인가
## 한 문장 요약
## 핵심 개념
- 용어: 쉬운 설명
## 쉬운 비유
## 이해 순서
1. ...
## 실무 체크리스트
- ...
## 원문에서 놓치면 안 되는 문장
- ...

규칙:
- 이모지 사용 금지
- 과장된 홍보 문구 금지
- 모르는 사실을 추가하지 말 것
- 원문에 없는 수치나 출처를 만들지 말 것
- SVG에 넣기 좋은 짧은 단계명 3~4개를 본문에 포함할 것

원문:
{{source_text}}
```

## 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 열어 사용합니다.

## 검증

```bash
npm run build
npm audit --omit=dev
```

## 설계 원칙

- 사용자가 정한 문서 맥락을 우선합니다.
- 화면 장식보다 인쇄 가독성을 우선합니다.
- SVG는 본문 내용을 보조하는 구조물로만 사용합니다.
- 사용자 입력은 HTML로 삽입하지 않고 텍스트로 렌더링합니다.
- 개인 로고나 특정 브랜드에 묶이지 않는 범용 오픈소스 도구로 유지합니다.
